UI Principles — Saia da Rotina

Objetivo

A interface deve transmitir acolhimento, calma e evolução.

O usuário nunca deve sentir que está utilizando um sistema administrativo.

Toda decisão visual deve reforçar a sensação de estar folheando a história da relação.

⸻

Hierarquia

Toda tela deve responder estas perguntas, nesta ordem:

1. Onde estou?
2. Como está nossa história?
3. O que merece destaque?
4. O que podemos fazer agora?

Nunca apresentar ações antes de contexto.

⸻

Densidade

Preferir poucas informações por tela.

Mais espaço em branco.

Menos elementos competindo por atenção.

Se houver dúvida entre adicionar ou remover um elemento, preferir remover.

⸻

Cartões

Cartões representam capítulos, momentos ou reflexões.

Nunca devem parecer tabelas.

Características:

* bordas suaves;
* cantos grandes;
* sombra discreta;
* bastante respiro interno;
* animações leves.

⸻

Cores

As cores possuem significado emocional.

Rosa

Romance

Carinho

Afeto

⸻

Roxo

Descobertas

Criatividade

Novas experiências

⸻

Verde

Evolução

Conexão

Crescimento

Nunca utilizar verde para transmitir produtividade.

⸻

Dourado

Celebrações

Conquistas

Momentos especiais

⸻

Tipografia

Títulos devem transmitir calma.

Evitar excesso de peso visual.

Corpo do texto deve ser confortável para leitura prolongada.

Nunca utilizar blocos enormes de texto.

⸻

Espaçamento

O espaço vazio faz parte do design.

Elementos importantes respiram.

Nunca comprimir cartões apenas para mostrar mais conteúdo.

⸻

Animações

Toda animação deve parecer natural.

Preferir:

fade

scale

opacity

slide curto

Evitar:

bounce

shake

flash

rotações exageradas

⸻

Ícones

Ícones representam emoções.

Preferir:

❤️

🌹

📖

✨

🤝

🕊️

🌙

☀️

🍷

💌

Evitar ícones corporativos como:

📊

📈

⚙️

🗂️

📋

Sempre que possível utilizar um conjunto consistente de ícones (ex.: Lucide) adaptado à identidade do produto.

⸻

Componentes

Todo componente deve possuir apenas uma responsabilidade.

Exemplos:

HeroCard

RelationshipCard

Progress

Badge

Section

EmptyState

Nunca criar componentes genéricos que misturem responsabilidades.

⸻

Estados vazios

Estados vazios são oportunidades para inspirar.

Nunca devem parecer erro.

Devem incentivar o próximo momento da história.

⸻

Responsividade

A experiência mobile é prioridade.

Desktop amplia a experiência.

Nunca o contrário.

⸻

Performance

Animações não podem comprometer fluidez.

Toda tela deve transmitir leveza.

A sensação percebida importa mais do que efeitos visuais complexos.

⸻

Princípio final

Se uma decisão visual não fortalece a sensação de estar vivendo uma história, ela provavelmente não pertence ao Saia da Rotina.
