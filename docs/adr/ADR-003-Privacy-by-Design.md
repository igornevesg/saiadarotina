ADR-004 — Living Book

Status: Aceito

Contexto

Os eventos registrados pelo sistema podem ser apresentados de diferentes formas.

Uma simples Timeline não representa o propósito do produto.

⸻

Decisão

A Timeline será tratada como infraestrutura.

A experiência apresentada ao usuário será chamada Living Book.

O Living Book poderá gerar:

* Timeline;
* Livro em PDF;
* Storytelling com IA;
* Memory Capsules;
* Relationship Weather.

Todos esses recursos utilizarão a mesma base de eventos.

⸻

Consequências

A camada de apresentação torna-se independente da camada de armazenamento.

Novas visualizações poderão ser criadas sem alterar o modelo de dados.
